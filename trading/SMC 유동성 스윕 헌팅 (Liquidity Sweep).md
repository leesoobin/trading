# 전략 개요 (Strategy Overview)

> **핵심 철학**
> 시장의 유동성 풀(Liquidity Pool)이 스윕(Sweep)되는 순간을 포착하여, **기관이 유동성을 확보한 직후 반대 방향으로 진입**한다. 스마트머니가 개인 트레이더의 손절을 터뜨린 후 진짜 방향으로 움직이는 구간에서 높은 RR을 확보하는 역추세/전환점 전략이다.

## 전략 성격
| 항목 | 값 |
|------|-----|
| 유형 | 추세 전환 / 역추세 (Reversal) |
| 목표 승률 | 30~40% |
| 목표 RR | 1:3 ~ 1:5+ |
| 매매 빈도 | 낮음 |
| 난이도 | 상 |
| 적합 장세 | **추세 전환점** (유동성 풀 도달 시) |

## 전략의 전제 조건
- HTF에서 **유동성 풀(EQH/EQL, 직전 고/저점)이 식별**되어야 함
- 유동성 스윕이 **HTF Supply/Demand Zone과 겹치는 구간**에서 발생해야 신뢰도 높음
- 스윕 후 **CHOCH가 발생**해야 추세 전환 시작으로 판단
- CHOCH만으로는 불충분 → **이후 BOS가 추세 전환을 확정**해야 진입
- 승률이 낮으므로 **높은 RR 확보가 필수** (최소 1:3)

# 1. 핵심 개념 정의

## 1-1. Liquidity (유동성)

### 유동성이란
- 시장에서 큰 주문을 가격 변동 없이 체결할 수 있는 능력
- 실전에서는 **개인 트레이더의 Stop Loss가 집중된 가격대** = 유동성 풀

### 유동성 풀이 형성되는 곳
| 위치 | 설명 | 스윕 방향 |
|------|------|----------|
| **EQH (Equal Highs)** | 비슷한 높이의 고점이 2~3회 형성 | 상방 스윕 후 하락 |
| **EQL (Equal Lows)** | 비슷한 높이의 저점이 2~3회 형성 | 하방 스윕 후 상승 |
| **직전 Swing High** | 명확한 직전 고점 | 상방 스윕 후 하락 |
| **직전 Swing Low** | 명확한 직전 저점 | 하방 스윕 후 상승 |
| **트렌드라인 위/아래** | 추세선을 따라 쌓인 손절 주문 | 추세선 이탈 스윕 후 복귀 |
| **라운드 넘버** | 심리적 가격대 (예: 100, 50000) | 터치 후 반전 |

### 외부 유동성 vs 내부 유동성
- **외부 유동성 (External)**: 구조의 최고점/최저점 너머에 쌓인 유동성
  - 스윕 시 **가장 높은 RR** 확보 가능 → 이 전략의 핵심 타깃
- **내부 유동성 (Internal)**: 구조 내부의 스윙 포인트에 쌓인 유동성
  - 외부 유동성을 놓쳤을 때 차선 진입 기회

## 1-2. Inducement (IDM)

### 정의
- 스마트머니가 **진짜 방향으로 움직이기 전에** 의도적으로 반대 방향으로 가격을 흔들어 개인 트레이더의 주문을 유도하는 행동

### IDM의 역할 (이 전략에서)
- IDM은 **스윕의 전조 현상**
- IDM이 발생하면 → 곧 유동성 스윕이 올 수 있다는 경고
- IDM → 스윕 → CHOCH 순서를 기다리는 인내심이 핵심

### IDM 식별
```
상승 추세에서:
    HH
   / \
  /   \  ← IDM (직전 고점 살짝 넘김)
 HL    LH ← 여기서 유동성 스윕 발생 가능
        \
         LL ← CHOCH (Strong Low 이탈)
```

## 1-3. Liquidity Sweep (유동성 스윕)

### 스윕의 정의
- 가격이 유동성 풀을 **일시적으로 돌파한 후 빠르게 되돌아오는** 현상
- 캔들의 **꼬리(wick)로만 찔렀다가** 몸통은 유동성 풀 안쪽에서 마감

### 스윕 유효성 판단
| 조건 | 유효한 스윕 | 무효한 스윕 |
|------|-----------|-----------|
| 캔들 마감 | 유동성 풀 안쪽에서 마감 | 유동성 풀 바깥에서 마감 (= BOS) |
| 꼬리 길이 | 긴 꼬리 (몸통 대비 2배 이상) | 짧은 꼬리 |
| HTF S/D | Supply/Demand Zone과 겹침 | 빈 구간에서 발생 |
| 후속 캔들 | 즉각적인 반전 움직임 | 추가 돌파 시도 |
| 거래량 | 스윕 캔들에서 거래량 급증 | 거래량 변화 없음 |

## 1-4. CHOCH → BOS 전환 확인

### 왜 CHOCH만으로 부족한가
- CHOCH는 **잠재적 추세 전환의 초기 신호**일 뿐
- CHOCH 이후 최소 2개의 고점 + 2개의 저점이 형성되어야 새로운 추세 채널 완성
- → **CHOCH 이후 BOS가 추가로 나타나야 완전한 반전 신호**

### 전환 확인 플로우
```
유동성 스윕 발생
    │
    ├── CHOCH 발생 (Strong High/Low 이탈)
    │       │
    │       ├── 되돌림 발생
    │       │       │
    │       │       └── BOS 발생 → 추세 전환 확정! → 진입 기회
    │       │
    │       └── 되돌림 없이 원래 방향 → CHOCH 실패 → 매매 보류
    │
    └── CHOCH 미발생 → 단순 스윕 후 추세 지속 → 매매 보류
```

# 2. 멀티타임프레임 (MTF) 분석 체계

## 2-1. 타임프레임 역할 분담

| 타임프레임 | 역할 | 확인 사항 |
|-----------|------|----------|
| **주봉/일봉 (HTF)** | 유동성 풀 + S/D Zone 식별 | EQH/EQL 위치, 대형 Supply/Demand Zone |
| **4시간봉 (HTF)** | 스윕 발생 감시 + 구조 전환 감지 | 스윕 캔들, CHOCH 발생, OB 식별 |
| **1시간봉 (MTF)** | 전환 구조 진행 확인 | CHOCH 후 BOS 확정 여부 |
| **15분/5분봉 (LTF)** | 정밀 진입 | OB/FVG에서 PA 확인, 진입 실행 |

## 2-2. 분석 플로우

```
[Step 1] HTF(일봉/주봉) 유동성 풀 식별
    │
    ├── EQH / EQL 위치 마킹
    ├── 직전 Swing High/Low 마킹
    ├── 해당 구간에 Supply/Demand Zone이 겹치는지 확인
    └── 유동성 풀 + S/D Zone 겹침 = 고확률 반전 구간
    │
[Step 2] HTF(4H) 스윕 발생 감시
    │
    ├── 가격이 유동성 풀에 접근
    ├── 스윕 캔들 발생 여부 확인 (긴 꼬리 + 안쪽 마감)
    └── 스윕 없이 BOS로 돌파 → 이 전략 무효, 추세추종으로 전환
    │
[Step 3] 구조 전환 확인 (4H → 1H)
    │
    ├── 스윕 후 CHOCH 발생 확인 (Strong High/Low 이탈)
    ├── CHOCH 후 되돌림 발생
    ├── BOS로 추세 전환 확정
    └── 전환이 확정된 OB 식별 (CHOCH를 만든 OB)
    │
[Step 4] LTF(15m/5m) 정밀 진입
    │
    ├── 전환 확정 후 OB로 되돌림 대기
    ├── OB 구간 내 PA 확인
    └── 진입 실행
```

# 3. 진입/손절/익절 규칙

## 3-1. 롱 진입 (하락 → 상승 전환)

### 진입 조건 체크리스트
- [ ] HTF(일봉+): 하방 유동성 풀 식별 (EQL, 직전 Swing Low, Demand Zone)
- [ ] HTF(4H): 유동성 풀이 **스윕됨** (긴 아래꼬리 + 안쪽 마감)
- [ ] HTF(4H): 스윕 구간이 HTF Demand Zone과 겹침 (컨플루언스)
- [ ] HTF(4H)/MTF(1H): 스윕 후 **Bullish CHOCH 발생** (직전 LH 상방 돌파)
- [ ] MTF(1H): CHOCH 후 되돌림 → **Bullish BOS 발생** (추세 전환 확정)
- [ ] BOS를 만든 Demand OB 식별 (Fresh 상태)
- [ ] 가격이 해당 OB 구간으로 되돌림
- [ ] LTF(5m/15m): OB 구간 내 Bullish PA 출현 (핀바/잉걸핑)
- [ ] RR 최소 1:3 이상 확보 가능

### 진입 시점
- **기본**: OB 구간 내 Bullish PA 캔들 종가 확정 시
- **공격적**: 스윕 캔들의 꼬리 끝에서 반등 시작 시 (CHOCH 전 진입 — 매우 위험하나 최고 RR)
- **보수적**: CHOCH + BOS 확정 후 OB 되돌림에서 PA 확인 시

### 손절
- **기본**: 스윕 캔들의 최저점(꼬리 끝) 하방
- 이 가격은 유동성이 이미 스윕된 구간이므로, 다시 이탈하면 스윕이 아닌 진짜 BOS
- 손절폭이 넓으므로 → **포지션 사이즈를 반드시 축소** (계좌 0.5~1%)

### 익절
- **1차 익절 (30%)**: 스윕 전 직전 LH (구조 내부) — RR 1:1~1:2
- **2차 익절 (30%)**: 스윕 전 직전 HH 또는 상방 유동성 풀 — RR 1:3
- **잔여 (40%) 트레일링**: 새로운 상승 구조의 BOS마다 손절 올림
  - 잔여분이 큰 이유: 추세 전환 초기에 잡았으므로 대파동을 탈 수 있음
- **최종 청산**: HTF에서 Bearish CHOCH 발생 시

## 3-2. 숏 진입 (상승 → 하락 전환)

### 진입 조건 체크리스트
- [ ] HTF(일봉+): 상방 유동성 풀 식별 (EQH, 직전 Swing High, Supply Zone)
- [ ] HTF(4H): 유동성 풀이 **스윕됨** (긴 위꼬리 + 안쪽 마감)
- [ ] HTF(4H): 스윕 구간이 HTF Supply Zone과 겹침 (컨플루언스)
- [ ] HTF(4H)/MTF(1H): 스윕 후 **Bearish CHOCH 발생** (직전 HL 하방 돌파)
- [ ] MTF(1H): CHOCH 후 되돌림 → **Bearish BOS 발생** (추세 전환 확정)
- [ ] BOS를 만든 Supply OB 식별 (Fresh 상태)
- [ ] 가격이 해당 OB 구간으로 되돌림
- [ ] LTF(5m/15m): OB 구간 내 Bearish PA 출현 (슈팅스타/잉걸핑)
- [ ] RR 최소 1:3 이상 확보 가능

### 진입 시점
- **기본**: OB 구간 내 Bearish PA 캔들 종가 확정 시
- **공격적**: 스윕 캔들의 꼬리 끝에서 하락 시작 시 (최고 RR, 최고 위험)
- **보수적**: CHOCH + BOS 확정 후 OB 되돌림에서 PA 확인 시

### 손절
- **기본**: 스윕 캔들의 최고점(꼬리 끝) 상방
- 손절폭이 넓으므로 → **포지션 사이즈 축소** (계좌 0.5~1%)

### 익절
- **1차 익절 (30%)**: 스윕 전 직전 HL (구조 내부) — RR 1:1~1:2
- **2차 익절 (30%)**: 스윕 전 직전 LL 또는 하방 유동성 풀 — RR 1:3
- **잔여 (40%) 트레일링**: 새로운 하락 구조의 BOS마다 손절 내림
- **최종 청산**: HTF에서 Bullish CHOCH 발생 시

# 4. 매매 보류 조건 (No Trade Zone)

다음 조건 중 하나라도 해당하면 **매매하지 않음**:

1. **유동성 풀이 식별되지 않음** — EQH/EQL 또는 명확한 Swing 포인트 없음
2. **스윕이 발생하지 않음** — 유동성 풀을 캔들 몸통으로 돌파 = BOS (전략 무효)
3. **HTF S/D Zone과 겹치지 않음** — 빈 구간에서의 스윕은 신뢰도 낮음
4. **CHOCH 미발생** — 스윕 후에도 기존 추세 구조 유지 중
5. **BOS 미확정** — CHOCH만 발생, 추세 전환이 아직 확정되지 않음
6. **RR 1:3 미만** — 이 전략은 높은 RR이 필수
7. **이미 추세 전환이 상당히 진행됨** — CHOCH + BOS + 추가 BOS 이후에는 전략 1(눌림목)로 전환
8. **주요 경제 이벤트 직전** — 예측 불가한 변동성

# 5. 시나리오 상세 (실전 플로우)

## 5-1. 롱 시나리오: 하방 유동성 스윕 후 상승 전환

### 시나리오 A: 외부 유동성 스윕 (최고 RR)
```
[전제] 일봉에서 주요 Swing Low 아래에 EQL 형성 + Demand Zone 존재
[발생] 4H에서 가격이 EQL 아래로 스윕 (긴 아래꼬리 + Demand Zone 내 마감)
[확인1] 4H/1H에서 Bullish CHOCH 발생 (직전 LH 상방 돌파)
[확인2] 되돌림 후 Bullish BOS 확정 (추세 전환 확정)
[마킹] BOS를 만든 Demand OB 식별
[대기] 가격이 Demand OB로 되돌림
[진입] LTF PA 확인 후 롱 진입
[손절] 스윕 캔들 최저점 하방
[익절] 1차 직전 LH → 2차 상방 유동성 풀 → 잔여 트레일링
[기대 RR] 1:3 ~ 1:5+
```

### 시나리오 B: 내부 유동성 스윕 (차선)
```
[전제] 외부 유동성 스윕을 놓친 상태
[발생] 전환 확정 후 상승 구조 진행 중, 내부 Swing Low를 스윕
[확인] 스윕 후 LTF에서 Bullish PA 출현
[진입] 내부 스윕 반등에서 롱 진입
[손절] 내부 스윕 최저점 하방
[기대 RR] 1:2 ~ 1:3
[주의] 외부 스윕 대비 RR이 낮으므로 포지션 사이즈 축소
```

## 5-2. 숏 시나리오: 상방 유동성 스윕 후 하락 전환

### 시나리오 A: 외부 유동성 스윕 (최고 RR)
```
[전제] 일봉에서 주요 Swing High 위에 EQH 형성 + Supply Zone 존재
[발생] 4H에서 가격이 EQH 위로 스윕 (긴 위꼬리 + Supply Zone 내 마감)

[확인2] 되돌림 후 Bearish BOS 확정 (추세 전환 확정)
[마킹] BOS를 만든 Supply OB 식별
[대기] 가격이 Supply OB로 되돌림
[진입] LTF PA 확인 후 숏 진입
[손절] 스윕 캔들 최고점 상방
[익절] 1차 직전 HL → 2차 하방 유동성 풀 → 잔여 트레일링
[기대 RR] 1:3 ~ 1:5+
```

### 시나리오 B: 내부 유동성 스윕 (차선)

```
[전제] 외부 유동성 스윕을 놓친 상태
[발생] 전환 확정 후 하락 구조 진행 중, 내부 Swing High를 스윕
[확인] 스윕 후 LTF에서 Bearish PA 출현
[진입] 내부 스윕 거부에서 숏 진입
[손절] 내부 스윕 최고점 상방
[기대 RR] 1:2 ~ 1:3
```

# 6. 유동성 스윕 vs 진짜 돌파(BOS) 구별법

이 전략의 **가장 중요한 판단 포인트**:

| 판단 기준 | 유동성 스윕 (이 전략 적용) | 진짜 돌파/BOS (전략 무효) |
|----------|----------------------|---------------------|
| **캔들 마감** | 유동성 풀 안쪽에서 마감 | 유동성 풀 바깥에서 마감 |
| **꼬리** | 긴 꼬리 (스윕 흔적) | 짧은 꼬리 or 마루보즈 |
| **후속 캔들** | 즉각 반전 (1~3 캔들 내) | 돌파 방향으로 추가 진행 |
| **거래량** | 스윕 캔들에서 스파이크 후 감소 | 돌파 후에도 거래량 유지/증가 |
| **HTF S/D** | S/D Zone 내에서 발생 | S/D Zone 없는 빈 구간 |
| **대응** | CHOCH 대기 → 진입 준비 | 추세추종(전략 1)으로 전환 |

# 7. AI 전문가 에이전트 디렉션

## 에이전트 역할 정의
> 이 에이전트는 "유동성 스윕 헌팅 전문가"로서, 유동성 풀 식별 → 스윕 감지 → 구조 전환 확인 → 고RR 진입의 순서로 분석하고 매매 판단을 제공한다.

## 분석 파이프라인 (순서대로 실행)

### Phase 1: 유동성 풀 식별 (HTF)
1. 일봉/주봉에서 EQH, EQL, 주요 Swing High/Low 식별
2. 해당 구간에 Supply/Demand Zone 겹침 여부 확인
3. 유동성 풀 + S/D Zone 컨플루언스 구간 마킹
4. → **식별 불가 시 "대기" 판정**

### Phase 2: 스윕 감지 (HTF → MTF)
5. 가격이 유동성 풀에 접근/도달 여부 확인
6. 스윕 캔들 발생 판단 (긴 꼬리 + 안쪽 마감)
7. 진짜 BOS인지 스윕인지 구별
8. → **스윕 미발생 또는 BOS 판정 시 "전략 무효" 판정**

### Phase 3: 구조 전환 확인 (MTF)
9. 스윕 후 CHOCH 발생 여부 확인
10. CHOCH 후 되돌림 → BOS 확정 여부 확인
11. → **CHOCH 미발생 또는 BOS 미확정 시 "대기" 판정**

### Phase 4: OB 진입 준비 (MTF → LTF)
12. 전환 확정 OB 식별 (CHOCH/BOS를 만든 OB)
13. 가격이 OB로 되돌림 중인지 확인
14. OB 구간 내 PA 탐색

### Phase 5: RR 검증 및 진입
15. 진입 가격, 손절 가격(스윕 극단점), 익절 목표가 산출
16. RR 계산 → **1:3 미만이면 패스**
17. 포지션 사이즈 산출 (계좌 0.5~1%)
18. 분할 익절 계획 (30% / 30% / 40% 트레일링)

### Phase 6: 최종 판단 출력
```
[판정] 롱/숏/보류
[신뢰도] 상/중/하
[유동성 풀] EQH/EQL/Swing High/Swing Low
[스윕 여부] 확인/미확인
[S/D 컨플루언스] 있음/없음
[구조 전환] CHOCH 확인 + BOS 확정 / CHOCH만 / 미확인
[OB 등급] S/A/B
[진입가] xxxx
[손절가] xxxx (스윕 극단점 기준)
[익절 1차] xxxx (R:R 1:x)
[익절 2차] xxxx (R:R 1:x)
[잔여 트레일링] 조건
[포지션 사이즈] 계좌의 x%
[주의사항] (해당 시)
```

# 8. 전략 한계 및 주의사항

1. **낮은 승률**: 30~40%이므로 연속 손실 5~7회를 심리적으로 견뎌야 함
2. **스윕 vs BOS 오판**: 스윕으로 판단했으나 진짜 돌파인 경우 → 손절 필수
3. **CHOCH 지연**: 스윕 후 CHOCH가 늦게 발생하면 진입가가 불리해짐
4. **넓은 손절**: 스윕 극단점까지의 거리가 크므로 포지션 사이즈 관리가 핵심
5. **과신 금지**: "이번엔 반드시 전환된다"는 생각은 금물 — 추세는 예상보다 오래 지속
6. **이벤트 리스크**: 뉴스 기반 스윕은 기술적 분석과 무관한 추가 변동 유발 가능

# 부록: 용어 정리

| 용어 | 설명 |
|------|------|
| Liquidity Pool | 손절 주문이 집중된 가격대 |
| EQH | Equal Highs — 비슷한 높이의 고점이 반복 형성된 구간 |
| EQL | Equal Lows — 비슷한 높이의 저점이 반복 형성된 구간 |
| Sweep | 유동성 풀을 일시적으로 돌파 후 빠르게 되돌아오는 현상 |
| Liquidity Grab | = Sweep, 기관이 유동성을 확보하는 행위 |
| CHOCH | Change of Character — 추세 전환 초기 신호 |
| BOS | Break of Structure — 추세 지속/전환 확정 |
| External Liquidity | 구조의 극단점 너머에 쌓인 유동성 (높은 RR) |
| Internal Liquidity | 구조 내부 스윙 포인트에 쌓인 유동성 |
| Confluence | 여러 근거가 겹치는 구간 (OB + S/D + 유동성 풀 등) |
